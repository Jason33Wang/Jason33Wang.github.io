现象

最开始现象

我似乎是不小心升级了某个模块导致了系统版本升级（但是升级不完全），最后导致，开机后进入不了桌面系统。
这时仅仅可以进入命令行界面ctrl+alt+{数字2-6}。登录后发现，wifi不能自动连网。

初步解决办法：

  • ifcongfig确认网卡是否正常工作，这个一般是正常的，如果不是，请到救援模式打开network，如果还不行，请刷机。
  • 利用nmcli dev wifi扫描wifi

  • 利用nmcli dev wifi connect [wifi的ssid] [wifi密码]连接wifi

    现在现象

    apt安装失败，显示无法解析域名。很显然命令行模式下，dns域名服务器没有设置。
    解决办法：

  • 修改vim /etc/resolv.conf文件，添加nameserver 8.8.8.8，保存退出（按esc，输入：wq，回车）。

  • 此时ping www.baidu.com,连接正常。

  • 这是一个临时修改办法，每次重启都需要重新设置。网上有一个永久修改方法，但是对我无效，我在此不做记录。

    现在现象

    网可以连接，直接修复系统即可。由于我可能是系统未正常升级导致的，所以我先重新升级。

    apt-get update
apt-get  upgrade

    中间可能会报一下错误：
    dpkg: error processing /var/cache/apt/archives/*.deb (--unpack):
    解决办法：强制执行
    sudo dpkg -i --force-overwrite /var/cache/apt/archives/*.deb

还可能报下面错误：

/usr/local/cuda-10.0/targets/x86_64-linux/lib//libcudnn.so.7 is not a symbolic link
解决办法是创建连接：

ln -sf /usr/local/cuda-10.0/targets/x86_64-linux/lib/libcudnn.so.7.4.2 /usr/local/cuda-10.0/targets/x86_64-linux/lib/libcudnn.so.7
请根据自己的情况（版本号）合理设置

安装更新后，reboot

现在现象：

可以开机登录，但是仍然进入不了桌面
解决办法

先确保不是 ubuntu-desktop的原因。
否则回到命令行界面
apt-get install ubuntu-desktop
reboot

现在现象：

可以登录，但是循环登录

解决方法;重新安装驱动
ubuntudrivers autoinstall
reboot

至此全部结束
